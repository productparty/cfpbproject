## Phase 2: Tableau Dashboard (Days 2-3)

Build in Tableau Cloud. Three views, one dashboard.

### View 1 — Complaint Volume Over Time
- **Type:** Line chart
- **X-axis:** Year-Month
- **Y-axis:** Count of complaints
- **Color:** Company_Short
- **Purpose:** Shows volume trends per company. Spikes = product issues or news events. Lets viewer compare scale (Chase will dwarf Chime — that's the point, normalize per-company if needed).

### View 2 — Issue Breakdown by Company
- **Type:** Stacked bar or heatmap
- **Rows:** Company_Short
- **Columns:** Issue (or Sub-issue for more detail)
- **Values:** Count of complaints
- **Purpose:** Reveals what each company's customers actually complain about. Chase might skew credit card fees; Chime might skew account access. Different complaint profiles = different product problems.

### View 3 — Response Quality Scorecard
- **Type:** Bar chart or KPI cards
- **Metric 1:** % Timely response by company
- **Metric 2:** Company response breakdown (Closed with explanation / Closed with monetary relief / Closed with non-monetary relief / Closed without relief)
- **Purpose:** "Who actually fixes things" — the product ops view. Shows whether companies resolve or deflect.

### Dashboard Assembly
- Combine all three views into a single dashboard
- Add a Company filter (multi-select) that controls all views
- Add a Date Range filter
- Add a Product filter
- Set dashboard size to fixed width that works well in an embed container

## Phase 3: Tableau Plus Features (Day 4)

### Pulse Metric Cards (set up in Tableau Cloud)
- **Metric 1:** Monthly complaint volume (all companies combined) — alert if month-over-month change > 20%
- **Metric 2:** Average timely response rate — alert if drops below 95%
- **Metric 3:** Monetary relief rate — % of complaints closed with monetary relief

### Simulated Agent (built in site/index.html)
A text input on the embedded page that maps keywords to Tableau filter actions.

**Pre-built queries:**
| User types... | Action |
|--------------|--------|
| "chase" or "Chase" | Filter Company_Short to Chase |
| "ally" or "Ally" | Filter Company_Short to Ally |
| "credit card" | Filter Product to credit card products |
| "checking" | Filter Product to checking/savings |
| "michigan" or "MI" | Filter State to MI |
| "2024" | Filter Year to 2024 |
| "compare chase ally" | Filter Company_Short to [Chase, Ally] |
| "reset" or "show all" | Clear all filters |

Implementation uses Tableau Embedding API v3 `applyFilterAsync()`.

## Key Decisions
- **Why CFPB data:** Real companies, real complaints, updated daily, government-published, free. Tells a genuine competitive story.
- **Why these 6 companies:** Mix of incumbent banks (Chase, Wells Fargo, Capital One) and digital-first fintechs (Ally, Chime, SoFi). Creates a natural comparison narrative.
- **Why 3 views:** Enough to tell a complete story (volume, issues, resolution) without overbuilding for a one-week sprint.
- **Why simulated Agent:** Demonstrates the Tableau Plus "agentic" concept without requiring actual NLP. Keyword matching is sufficient for a demo.
